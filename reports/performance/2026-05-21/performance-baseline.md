## Stack Discovery

- Framework: none detected.
- Rendering mode: static HTML served by GitHub Pages.
- Build system: none detected; no `package.json`, lockfile, Vite, Next, Astro, Netlify, or Vercel config.
- Routing model: static files at `/` (`index.html`) and `/fernando.html`; in-page anchors for homepage sections.
- Deployment assumptions: GitHub Pages via `CNAME` with apex domain `ferariza.com`.
- Image handling: direct PNG/JPG/GIF/SVG files under `assets/images`; no responsive image pipeline.
- Font loading: local icon font stylesheets for ET Line Icons, Linea, and Font Awesome; no text webfont dependency except `assets/js/vendor/google-fonts.js`.
- CSS strategy: multiple blocking stylesheets plus a large legacy `assets/styles/main.css` file.
- JavaScript strategy: jQuery-era plugin stack loaded on every page, plus Google Analytics and Twitter widget.
- Analytics/third-party: Google Analytics UA property (`UA-152728016-1`) and `platform.twitter.com/widgets.js`.
- Sitemap/robots: not present locally or on production at baseline.
- Metadata: basic title, description, viewport, favicon; missing canonical, Open Graph, and Twitter card tags.
- Structured data: none found.

## Key Findings

### Performance

- Homepage mobile LCP is dominated by the Slick hero slide background: `assets/images/lluvia.png` is a 2.3 MB PNG and is the LCP element.
- Homepage book covers are very large PNGs: `fuiste-el-rey.png` is 1.47 MB and `ciudad-dormida.png` is 1.41 MB. Lighthouse flags both modern format and responsive-size savings.
- `assets/styles/main.css` is 690 KB locally, mostly unused on first load; production compression reduces transfer size but the CSS is still render-blocking.
- Multiple CSS files block rendering: Bootstrap, icon fonts, Slick, Magnific Popup, Animate.css, `main.css`, and `responsive.css`.
- `modernizr-2.8.3.min.js` is loaded synchronously in the head and is render-blocking on mobile.
- Google Analytics and Twitter add third-party work; Twitter also contributes console/DevTools issues and third-party cookie warnings.

### Accessibility

- Homepage slider buttons generated by Slick lack accessible names.
- Icon-only social and gallery links lack accessible labels.
- Some decorative/slider DOM states create `aria-hidden` focusable descendant failures.
- Color contrast failures exist in legacy theme elements.
- Heading order is not sequential on both pages.
- Some client/logo list markup is invalid after the Twitter widget/script mutation.

### Best Practices

- Production has third-party cookie warnings from third-party embeds.
- Homepage logs a `customTwitter()` console error because the Twitter iframe is not guaranteed to exist at `window.onload`.
- Local Lighthouse best-practices scores are lower because local HTTP is not HTTPS; production HTTPS is correct.

## Baseline Improvement Targets

P0:

- Replace oversized above-the-fold and book-cover PNG delivery with responsive, modern image formats.
- Remove or defer render-blocking JavaScript.
- Fix accessibility names for generated controls and icon-only links.
- Remove the Twitter `customTwitter()` runtime error.
- Add robots, sitemap, canonical URLs, social metadata, and structured data.

P1:

- Lazy-load below-the-fold images.
- Add explicit image dimensions to reduce future CLS risk.
- Reduce unused CSS/JS where safely possible without redesigning the legacy theme.
- Keep analytics and visible social/profile links intact.
